> Stage 0 결과가 완벽하지 않은 이유는 간단합니다. **AI는 업무 맥락을 모르기 때문입니다.** 여러분이 알고 있는 것을 구체적으로 알려주면 결과가 빠르게 좋아집니다.

| :material-timer-outline: 예상 소요 | :material-puzzle-outline: 핵심 작업 | :material-star-outline: 난이도 |
|:--:|:--:|:--:|
| 20분 | 규칙, 위치, 용어, 예외 전달 | 낮음~보통 |

!!! abstract "이 단계의 핵심"
    - 단순히 "틀렸어"라고 하지 않는다
    - 어떤 규칙을 놓쳤는지 설명한다
    - 피드백 → 수정 → 확인을 최소 2회 이상 반복한다

!!! tip "이 Stage의 본질 = 관찰담 전달하기"
    Stage 0에서 메모한 "이상한 점"들을 AI에게 구체적으로 설명하는 것이 이 Stage의 전부다. 용어 교정, PDF 위치, 표 구조, 업무 규칙, 예외 상황 — 전부 관찰담의 종류일 뿐이다.

## 1-1. 부족한 부분을 짚어주기

### AI에게 줄 수 있는 도메인 지식

| 종류 | 무엇을 알려주나 | 예시 |
|------|----------------|------|
| 용어 교정 | 비슷한 표현 중 어떤 말이 정답인지 | "XX"가 아니라 "YY"가 맞아 |
| PDF 내 위치 | 몇 페이지, 어떤 표, 어떤 구역인지 | 이 정보는 [N]페이지 [M]번째 표에 있어 |
| 표 구조 | 셀 병합, 페이지에 걸친 표, 다단 헤더 여부 | 이 표는 다음 페이지까지 이어져 |
| 업무 규칙 | 빈칸 처리, 날짜 형식, 없음 처리 기준 | 최저학력기준이 없으면 "없음"이 아니라 빈칸이야 |
| 예외 상황 | 건너뛰어야 하는 페이지나 함정 | 이 페이지는 목차니까 넘어가 |

!!! example "이런 식으로 말해보세요"
    ```text
    1. J(전형유형)랑 K(전형구분)이 뒤바뀌어 있어
    지금 J 컬럼에 "지역균형전형", "일반전형" 이렇게 들어가 있고, 
    K 컬럼에는 전부 "학생부종합"이 들어가 있거든? 이거 반대야.
    
    J(전형유형) = 학생부종합, 학생부교과, 논술 같은 전형의 종류
    K(전형구분) = 지역균형, 일반, 사회통합 같은 전형 내 세부 구분

    2. 1단계 행이 통째로 없어 — 209행밖에 안 돼
    정답은 418행인데 지금 209행이야. 정확히 절반이지? 이유는 1단계 
    행을 안 만들었기 때문이야. 서울대는 거의 모든 전형이 1단계(서류) 
    → 2단계(면접) 구조야. 모집단위 하나당 2행을 만들어야 해.

    3. AA(사정모형)에 설명을 넣으면 안 돼
    지금 AA에 "1단계(서류평가, 3배수)" 이런 식으로 넣었는데, 
    이 칸은 그냥 "1단계" 또는 "2단계"만 넣는 칸이야.

    4. AC(사정비율)가 전부 비어 있어
    1단계 행이면 배수 × 100 (3배수→300, 2배수→200), 
    2단계 행이면 100을 넣어.

    5. 면접(AE)·서류(AH) 배점이 단계 구분 없이 들어가 있어
    1단계: 서류 100, 면접 빈칸
    2단계: 서류 70, 면접 30

    6. X(수능응시영역)·Y(최저학력기준) — PDF 원문 그대로 넣지 마
    정답 예시: 국어,수학[확/미/기],영어,사탐/과탐(2),한국사
    최저가 없는 전형은 빈칸으로 둬.

    7. W(지원자격) — 전형구분별로 다른 텍스트를 넣어야 해
    지역균형은 "고교장 추천", 일반은 "고교 졸업자 또는 동등학력자"
    ```

## 1-2. 판단이 들어가는 정보 채워 넣기 (선택)

자동화 작업 중 명확한 분류 체계가 있는데 PDF 내용을 보고 분류 작업이 필요한 경우가 있습니다. 여기에선 PART 컬럼이 그 중 하나입니다.

- `1` = 인문
- `2` = 자연  
- `3` = 예체능
- `4` = 자유

국어국문학과나 중어중문학과는 1, 공과대학의 학부들은 2로 분류합니다. 농업생명과학대학은 모두 2로 보이지만 농경제사회학부는 1로 분류되어야 합니다.

!!! example "PART 컬럼 분류 작업 지시 예시"
    ```text
    PART 컬럼을 채워넣기 위해선 판단이 필요해. COLL과 MajorName을 
    보고 1,2,3,4 중 하나로 판단해야해. COLL과 MajorName의 조합을 
    반드시 확인하고 분류해야해. 예를 들어, 농업생명과학대학의 대부분의 
    학부는 자연이지만 농경제사회학부는 인문이야.
    ```

## 1-3. 결과 확인하고 다시 피드백하기

!!! success "반복 루프"
    1. 수정된 결과를 확인한다
    2. 아직 부족한 부분을 다시 짚는다
    3. 어떤 규칙이 빠졌는지 더 명확하게 말한다
    4. 결과가 만족스러워질 때까지 반복한다 (실습에선 1번만!)

!!! example "다시 확인 요청하는 문장 예시 (1번만 반복해보기)"
    ```text
    수정한 결과를 다시 보여줘.
    그리고 아직 부족한 부분이 있으면 내가 다시 피드백할게.
    이번에는 특히 [전형유형/전형구분], 날짜 형식, 빈칸 처리 규칙을 신경 써줘.
    ```

!!! tip "좋은 피드백 팁"
    - `전형유형`과 `전형구분`처럼 의미가 비슷한 컬럼은 서로 뒤바뀌기 쉽습니다
    - 행 수가 예상의 절반이면 구조가 통째로 누락됐는지 의심할 수 있습니다
    - 날짜는 `YYYY-MM-DD`처럼 출력 형식을 명시하는 편이 좋습니다

??? warning "잘 안 될 때"
    - 표 추출이 계속 부정확하면 먼저 `pdfplumber`, `PyMuPDF` 같은 PDF 라이브러리를 바꿔볼 수 있습니다
    - 그래도 텍스트가 거의 안 뽑히면 이미지형 PDF일 수 있으므로, 다음 Stage 설계에 OCR이나 Vision API 경로를 넣으면 됩니다
