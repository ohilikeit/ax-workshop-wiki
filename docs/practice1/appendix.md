    직접 해석하려고 애쓰기보다, 에러 메시지를 그대로 복사해서 AI에게 전달하면 됩니다.

---

## 에러가 났을 때

!!! example "이 문장을 그대로 사용하세요"
    ```text
    이런 에러가 났어:
    [에러 메시지를 그대로 복사해서 붙여넣기]

    원인이 뭔지 설명해주고 해결해줘.
    ```

!!! success ""
    핵심은 에러를 이해하는 것이 아니라, **있는 그대로 전달하는 것**입니다. AI가 원인과 해결책을 정리해주면 그 흐름을 따라가면 됩니다.

---

## 자주 쓰는 개념

!!! abstract "일반화"
    지금 PDF에만 맞는 규칙이 아니라, 다른 PDF에도 적용되는 구조로 만드는 것

!!! abstract "검증 루프"
    결과 확인 → 오류 수정 → 재검증을 반복하는 것

!!! abstract "셀 병합 처리"
    위 칸의 값이 아래 여러 줄에 적용되는 구조를 해석하는 것

!!! abstract "대체 경로"
    기본 PDF 추출이 안 될 때 OCR이나 Vision API 같은 다른 방법으로 넘기는 흐름

??? note "실습하면서 추가하면 좋은 자료"
    - 프롬프트 입력 화면
    - AI 응답 예시 화면
    - Excel 결과물 예시
    - 에러 발생 전후 화면
    - 사람이 직접 확인하는 비교 화면
