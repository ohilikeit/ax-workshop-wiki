    부록. 에러 대처와 프롬프트 팁¶

    ← 메인으로 실습 1 개요

    실습 중 빨간 글씨 에러가 나와도 정상입니다

    직접 해석하려고 애쓰기보다, 에러 메시지를 그대로 복사해서 AI에게 전달하면 됩니다.

    에러가 났을 때¶

    AI에게 이렇게 말해보세요 — 에러 메시지 전달

    이런 에러가 났어:
[에러 메시지를 그대로 복사해서 붙여넣기]

원인이 뭔지 설명해주고 해결해줘.

    핵심은 에러를 이해하는 것이 아니라, 있는 그대로 전달하는 것입니다. AI가 원인과 해결책을 정리해주면 그 흐름을 따라가면 됩니다.

    자주 쓰는 개념¶

    일반화

    지금 PDF에만 맞는 규칙이 아니라, 다른 PDF에도 적용되는 구조로 만드는 것입니다

    검증 루프

    결과 확인 → 오류 수정 → 재검증을 반복하는 것입니다

    셀 병합 처리

    위 칸의 값이 아래 여러 줄에 적용되는 구조를 해석하는 것입니다

    대체 경로

    기본 PDF 추출이 안 될 때 OCR이나 Vision API 같은 다른 방법으로 넘기는 흐름입니다

    실습하면서 추가하면 좋은 자료
    • 프롬프트 입력 화면
    • AI 응답 예시 화면
    • Excel 결과물 예시
    • 에러 발생 전후 화면
    • 사람이 직접 확인하는 비교 화면

    이 부록은 실습 중 막혔을 때 빠르게 돌아와서 참고하는 페이지입니다. 메인 가이드와 각 Stage 페이지에서 필요한 순간에 같이 열어두면 좋습니다.
